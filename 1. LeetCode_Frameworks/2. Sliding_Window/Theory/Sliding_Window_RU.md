# Фреймворк Sliding Window

## 1. Где это находится в общей картине

LeetCode удобно изучать не как набор случайных задач, а как систему:

- **11 фреймворков**

- второй фреймворк: **Sliding Window**

Главная идея: не зубрить отдельные решения, а учиться **распознавать форму задачи** и быстро выбирать подходящий метод.

---

## 2. Что такое Sliding Window

**Sliding Window** — это подход, где мы рассматриваем **группу соседних элементов** массива или строки и двигаем эту группу вправо, обновляя ответ эффективно, без пересчёта с нуля.

Цель подхода:

- избежать вложенных циклов;
- решить задачу за **O(n)** вместо **O(n²)**;
- работать с **подотрезками** и **подстроками**.

---

## 3. Когда этот фреймворк обычно встречается

Sliding Window часто появляется, когда:

- нужно найти что-то среди **подряд идущих элементов**;
- есть ограничение на **длину** или **состав** подотрезка;
- нужно найти **максимальный / минимальный / лучший** подотрезок;
- данные — массив или строка.

---

## 4. Главная схема распознавания

```text
Sliding Window
│
├── Длина окна фиксирована?
│   └── Да → Fixed-size window
│
└── Длина окна не фиксирована
    ├── Элементы разбиваются на непересекающиеся группы?
    │   └── Да → Non-overlapping windows
    │
    └── Нужно найти лучший подотрезок по условию?
        └── Да → Variable-size window
```

### Вопросы, которые надо задавать себе

1. Длина окна **фиксирована** (дано `k`)?
2. Массив разбивается на **отдельные группы**, каждый элемент в одной группе?
3. Нужно найти **оптимальный подотрезок** с изменяемой длиной?

---

## 5. Основные типы Sliding Window

### 5.1. Fixed-size window

#### Идея

Если нужно найти характеристику у **`k` подряд идущих элементов**, используем окно фиксированной длины.

Сначала считаем сумму первого окна длины `k`.
Потом двигаем окно вправо:
- вычитаем элемент, который ушёл слева;
- добавляем элемент, который пришёл справа.

Переход к следующему окну — за **O(1)**.

#### Когда использовать

- максимальная сумма `k` подряд идущих элементов;
- среднее значение для каждого окна длины `k`;
- количество подстрок фиксированной длины.

#### Ключевая формула перехода

`новое_окно = старое_окно - левый_элемент + правый_элемент`

#### Пример — LeetCode 643

```python
class Solution:
    def findMaxAverage(self, nums, k):
        window_sum = 0

        for i in range(k):
            window_sum += nums[i]
        max_sum = window_sum

        for r in range(k, len(nums)):
            left = r - k
            window_sum = window_sum + nums[r] - nums[left]
            max_sum = max(max_sum, window_sum)

        return max_sum / k
```

#### Сложность

- **Время:** `O(n)`
- **Память:** `O(1)`

#### Green flags

- в условии есть число `k`;
- нужно рассмотреть все окна одной длины;
- нужна сумма, среднее или другая агрегация по подотрезку.

---

### 5.2. Non-overlapping windows

#### Идея

Если массив естественно разбивается на **группы**, и каждый элемент принадлежит **ровно одной группе**, используем непересекающиеся окна.

Два указателя:
- `l` — начало текущей группы;
- `r` — конец текущей группы.

Двигаем `r` вправо, пока условие группы выполняется. Обрабатываем окно `[l, r]`, затем переходим к следующей группе: `l = r + 1`.

#### Когда использовать

- сжать подряд идущие числа в диапазоны;
- разбить массив на группы по условию;
- обработать блоки одинаковых или последовательных элементов.

#### Пример

```python
def compress_ranges(nums: list[int]) -> list[str]:
    l = 0
    r = 0
    result = []

    while l < len(nums):
        while r + 1 < len(nums) and nums[r] + 1 == nums[r + 1]:
            r += 1

        if r != l:
            result.append(f"{nums[l]}->{nums[r]}")
        else:
            result.append(f"{nums[l]}")

        l = r + 1
        r = r + 1

    return result
```

#### Сложность

- **Время:** `O(n)` — оба указателя двигаются только вперёд
- **Память:** `O(n)` — если ответ хранится в отдельном массиве

#### Green flags

- группы идут одна за другой;
- каждый элемент принадлежит ровно одной группе;
- группы не пересекаются.

### 5.3. Variable-size window

#### Идея

Если нужно найти **лучший подотрезок** или **лучшую подстроку**, удовлетворяющую некоторому условию, используем окно переменной длины.

Два указателя:
- `l` — левая граница окна;
- `r` — правая граница окна.

Стараемся **максимально расширить окно вправо**.
Когда расширять нельзя — обновляем ответ и сужаем окно слева.

#### Главное — выбрать состояние окна

**Состояние окна** — это величина, которая описывает текущий подотрезок и помогает быстро понять: можно ли расширить окно или нужно его сузить.

Например, если можно заменить не более `k` нулей на единицы — состояние окна это `zeros_count`.

#### Когда использовать

- самая длинная подстрока без повторяющихся символов;
- максимальная длина отрезка с не более `k` плохими элементами;
- минимальный подотрезок, удовлетворяющий условию.

#### Пример — LeetCode 3

```python
class Solution:
    def lengthOfLongestSubstring(self, s):
        char_set = set()
        l = 0
        result = 0

        for r in range(len(s)):
            while s[r] in char_set:
                char_set.remove(s[l])
                l += 1
            char_set.add(s[r])
            result = max(result, r - l + 1)

        return result
```

#### Сложность

- **Время:** `O(n)` — оба указателя двигаются только вправо
- **Память:** `O(1)`

#### Green flags

- нужно найти максимальный / минимальный / лучший подотрезок;
- есть условие, которому должно удовлетворять окно;
- длина окна изменяется.

- arrays

### Почему это важно

**Arrays**
- индексация;
- работа с границами;
- проход по элементам.

**Two Pointers**
- движение двух указателей;
- понимание левой и правой границы;
- сужение и расширение диапазона.

**Hash Maps**
- подсчёт частоты символов;
- быстрая проверка условия окна;
- нужны в задачах на подстроки.

## 7. Признаки, что перед тобой задача на Sliding Window

### Частые слова в условии

- `subarray`
- `substring`
- `contiguous`
- `window`
- `k elements`
- `longest` / `shortest`
- `maximum` / `minimum`

### На что реагировать

- `at most k` — не более `k` чего-либо
- `without repeating` — без повторений
- `consecutive elements` — подряд идущие элементы
- `fixed length` — фиксированная длина

---

## 8. Частые ошибки

1. **Неправильное состояние окна**
   Выбрана не та переменная для описания текущего окна.

2. **Забыли обновить состояние при сужении**
   При сдвиге `l` вправо состояние окна не обновляется.

3. **Неправильная инициализация**
   Особенно актуально для variable-size: `r = -1` нужен, чтобы окно начинало пустым.

4. **Путаница Fixed-size и Variable-size**
   Если в задаче есть `k` — скорее всего fixed-size. Если нет — variable-size.

5. **Лишние пересчёты**
   Считать всё окно заново вместо использования формулы перехода.

## 9. Быстрые шаблоны

### Fixed-size window

```python
window = sum(nums[:k])
best = window

for r in range(k, len(nums)):
    l = r - k
    window = window + nums[r] - nums[l]
    best = max(best, window)
```

### Non-overlapping windows

```python
l = 0
r = 0

while l < len(nums):
    while r + 1 < len(nums) and condition(r):
        r += 1

    # обработать окно [l, r]

    l = r + 1
    r = r + 1
```

### Variable-size window

```python
l = 0
r = -1
result = 0

while l < len(nums):
    while r + 1 < len(nums) and can_expand(r + 1):
        update_state(r + 1)
        r += 1

    result = max(result, r - l + 1)

    remove_from_state(l)
    l += 1
```

## 10. Итог

### Sliding Window — это семейство подходов

1. **Fixed-size window**
   - длина окна фиксирована (`k`)
   - формула перехода: `новое = старое - левый + правый`

2. **Non-overlapping windows**
   - массив разбивается на непересекающиеся группы
   - каждый элемент в одной группе

3. **Variable-size window**
   - длина окна меняется
   - нужно выбрать правильное состояние окна

### Главная мысль

Сложность sliding window не в циклах.
Главное — правильно выбрать **состояние окна** и понять:
- когда расширять;
- когда сужать;
- как обновлять состояние при каждом движении.
