# 912. Sort an Array

Документация к задаче LeetCode 912 с разбором двух решений на Python из этой папки:

- `912(1).py` - простой и наглядный `merge sort`
- `912(2).py` - более оптимизированный `merge sort` с общим временным буфером

---

## Условие задачи

Нужно отсортировать массив `nums` по возрастанию и вернуть отсортированный результат.

Пример:

```text
Вход:  [5, 2, 3, 1]
Выход: [1, 2, 3, 5]
```

---

## Почему здесь подходит Merge Sort

`Merge sort` хорошо подходит для этой задачи, потому что:

- гарантирует `O(n log n)` по времени
- стабильно работает даже на больших входных данных
- легко разбивается на рекурсивные шаги
- хорошо соответствует формату задач LeetCode

Идея алгоритма простая:

1. Разделить массив на две половины
2. Отсортировать каждую половину отдельно
3. Слить две отсортированные части в один итоговый массив

---

## Решение 1: `912(1).py`

Это классическая учебная версия `merge sort`.

```python
class Solution(object):
    def sortArray(self, nums):
        def merge_sort(arr):
            if len(arr) <= 1:
                return arr

            mid = len(arr) // 2

            left = merge_sort(arr[:mid])
            right = merge_sort(arr[mid:])

            return merge(left, right)

        def merge(left, right):
            result = []
            i = 0
            j = 0

            while i < len(left) and j < len(right):
                if left[i] <= right[j]:
                    result.append(left[i])
                    i += 1
                else:
                    result.append(right[j])
                    j += 1

            while i < len(left):
                result.append(left[i])
                i += 1

            while j < len(right):
                result.append(right[j])
                j += 1

            return result

        return merge_sort(nums)
```

### Как работает

1. Если в массиве `0` или `1` элемент, он уже отсортирован
2. Массив делится на левую и правую части
3. Обе части сортируются рекурсивно
4. Функция `merge()` объединяет две отсортированные части

### Плюсы

- очень легко читать и объяснять
- хорошо подходит для изучения идеи `merge sort`

### Минусы

- используются срезы `arr[:mid]` и `arr[mid:]`
- на каждом уровне рекурсии создаются новые списки
- в Python это даёт дополнительную нагрузку на память и копирование данных

## Решение 2: `912(2).py`

Это более практичная и быстрая версия того же алгоритма.

```python
class Solution(object):
    def sortArray(self, nums):
        n = len(nums)
        temp = [0] * n

        def merge_sort(left, right):
            if left >= right:
                return

            mid = (left + right) // 2

            merge_sort(left, mid)
            merge_sort(mid + 1, right)

            if nums[mid] <= nums[mid + 1]:
                return

            merge(left, mid, right)

        def merge(left, mid, right):
            i = left
            j = mid + 1
            k = left

            while i <= mid and j <= right:
                if nums[i] <= nums[j]:
                    temp[k] = nums[i]
                    i += 1
                else:
                    temp[k] = nums[j]
                    j += 1
                k += 1

            while i <= mid:
                temp[k] = nums[i]
                i += 1
                k += 1

            while j <= right:
                temp[k] = nums[j]
                j += 1
                k += 1

            for p in range(left, right + 1):
                nums[p] = temp[p]

        merge_sort(0, n - 1)
        return nums
```

### Чем это решение лучше

- не создаёт новые подмассивы через срезы
- работает по индексам `left` и `right`
- использует один общий массив `temp`
- умеет пропускать слияние, если две соседние части уже упорядочены

Проверка:

```python
if nums[mid] <= nums[mid + 1]:
    return
```

означает, что левая половина уже не больше правой на границе, а значит текущее слияние можно не делать.

### Как работает

1. Рекурсивно сортируется диапазон `left..right`
2. После сортировки двух половин вызывается `merge()`
3. В `merge()` элементы сравниваются и временно записываются в `temp`
4. После этого отсортированный диапазон копируется обратно в `nums`

---

## Пример работы Merge Sort

Возьмём массив:

```text
[5, 2, 3, 1]
```

Разбиение:

```text
[5, 2]   [3, 1]
```

После рекурсивной сортировки:

```text
[2, 5]   [1, 3]
```

После финального слияния:

```text
[1, 2, 3, 5]
```

## Сложность

### `912(1).py`

- Время: `O(n log n)`
- Память: больше классического `O(n)` из-за срезов и создания новых списков

### `912(2).py`

- Время: `O(n log n)`
- Память: `O(n)` за счёт общего массива `temp`

---

## Когда использовать каждую версию

- `912(1).py` стоит использовать, если важна простота и наглядность
- `912(2).py` лучше использовать, если важнее эффективность и ближе к реальному интервью/сабмиту

---

## Ключевая идея задачи

Главная сила `merge sort` в том, что мы не пытаемся сразу отсортировать весь массив целиком. Мы:

- дробим задачу на маленькие части
- решаем каждую часть отдельно
- объединяем уже отсортированные результаты

Это и даёт устойчивую сложность `O(n log n)`.
