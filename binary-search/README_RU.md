# Binary Search

Чистое и понятное объяснение алгоритма бинарного поиска на основе решения в формате LeetCode.

---

## Что такое Binary Search?

**Binary Search (бинарный поиск)** — это алгоритм поиска элемента в **отсортированном массиве**.

Вместо того чтобы проверять элементы по одному, он каждый раз смотрит на **середину текущего диапазона** и отбрасывает половину, в которой нужного элемента точно быть не может.

Именно поэтому он работает гораздо быстрее обычного линейного поиска.

---

## Главное условие

Бинарный поиск работает только тогда, когда массив **отсортирован по возрастанию**.

Пример:

```text
[-1, 0, 3, 5, 9, 12]
```

---

## Основная идея

Мы храним две границы поиска:

- `low` — левая граница
- `high` — правая граница

На каждом шаге:

1. Находим индекс середины
2. Сравниваем значение в середине с `target`
3. Если равны — возвращаем индекс
4. Если значение в середине больше — идём влево
5. Если меньше — идём вправо

---

## Код на Python (формат LeetCode)

```python
class Solution:
    def search(self, nums, target):
        low = 0
        high = len(nums) - 1

        while low <= high:
            mid = (low + high) // 2
            guess = nums[mid]

            if guess == target:
                return mid

            if guess > target:
                high = mid - 1
            else:
                low = mid + 1

        return -1
```

---

## Разбор кода

### `low = 0`
Левая граница начинается с первого индекса массива.

### `high = len(nums) - 1`
Правая граница начинается с последнего допустимого индекса.

### `while low <= high`
Пока диапазон поиска существует, продолжаем искать.

### `mid = (low + high) // 2`
Находим индекс середины текущего диапазона.

### `guess = nums[mid]`
Берём значение по индексу `mid`.

### `if guess == target`
Если нашли нужный элемент, сразу возвращаем его индекс.

### `if guess > target`
Если значение в середине больше `target`, значит нужный элемент может быть только **слева**.

Поэтому:

```python
high = mid - 1
```

### `else`
Если значение в середине меньше `target`, значит нужный элемент может быть только **справа**.

Поэтому:

```python
low = mid + 1
```

### `return -1`
Если цикл закончился и элемент не найден, возвращаем `-1`.

Именно это требует задача LeetCode 704.

---

## Пример работы

Массив:

```text
[-1, 0, 3, 5, 9, 12]
```

Ищем:

```text
9
```

### Шаг 1
- `low = 0`
- `high = 5`
- `mid = (0 + 5) // 2 = 2`
- `guess = nums[2] = 3`

`9 > 3`, значит идём вправо.

### Шаг 2
- `low = 3`
- `high = 5`
- `mid = (3 + 5) // 2 = 4`
- `guess = nums[4] = 9`

Элемент найден.

Ответ: `4`

---

## Почему это быстро?

Обычный поиск проверяет элементы по одному.

Бинарный поиск каждый раз **делит область поиска пополам**.

Примеры:

- `100` элементов → около `7` шагов
- `1 000` элементов → около `10` шагов
- `1 000 000` элементов → около `20` шагов
- `1 000 000 000` элементов → около `30` шагов

---

## Сложность

**Временная сложность:** `O(log n)`

Это значит, что даже при сильном росте массива число операций растёт очень медленно.

Бинарный поиск значительно быстрее линейного поиска `O(n)` на больших отсортированных данных.

---

## Что нужно запомнить

- массив должен быть **отсортирован**
- поиск идёт по **индексам**
- на каждом шаге берётся **середина диапазона**
- если `target` меньше середины — идём влево
- если `target` больше середины — идём вправо
- если нашли элемент — возвращаем индекс
- если не нашли — возвращаем `-1`

---

## Вывод

Бинарный поиск — это один из самых важных базовых алгоритмов.

Он учит простой, но мощной идее:

> Не нужно проверять всё подряд, если можно сразу отбросить половину неправильных вариантов.

Именно поэтому алгоритм получается быстрым, красивым и очень полезным в задачах на программирование.
