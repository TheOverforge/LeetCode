# Binary Search

## Что это такое

**Binary Search (бинарный поиск)** — это алгоритм поиска элемента в **отсортированном массиве**.  
Он работает быстрее обычного перебора, потому что на каждом шаге **отбрасывает половину массива**.

---

## Главное условие

Бинарный поиск работает **только с отсортированными данными**.

Пример подходящего массива:

[1, 3, 5, 7, 9]

---

## Идея алгоритма

На каждом шаге алгоритм:

1. Берёт левую границу.
2. Берёт правую границу.
3. Находит середину.
4. Сравнивает значение в середине с искомым числом.
5. Если нашли — возвращает индекс.
6. Если искомое число меньше — поиск продолжается в левой части.
7. Если больше — в правой части.

---

## Важно понимать

Бинарный поиск делит пополам **не сами значения**, а **диапазон индексов**.

То есть:

- `low` — левый индекс
- `high` — правый индекс
- `mid = (low + high) // 2` — средний индекс
- `guess = list[mid]` — значение по этому индексу

---

## Пример

Массив:

индекс:   0  1  2  3  4  
значение: 1  3  5  7  9

Ищем число `3`.

### Шаг 1
- `low = 0`
- `high = 4`
- `mid = (0 + 4) // 2 = 2`
- `guess = list[2] = 5`

`3 < 5`, значит ищем в левой части.

### Шаг 2
- `low = 0`
- `high = 1`
- `mid = (0 + 1) // 2 = 0`
- `guess = list[0] = 1`

`3 > 1`, значит ищем в правой части.

### Шаг 3
- `low = 1`
- `high = 1`
- `mid = (1 + 1) // 2 = 1`
- `guess = list[1] = 3`

Элемент найден. Ответ: индекс `1`.

---

## Код на Python

```python
def binary_search(lst, item):
    low = 0
    high = len(lst) - 1

    while low <= high:
        mid = (low + high) // 2
        guess = lst[mid]

        if guess == item:
            return mid

        if guess > item:
            high = mid - 1
        else:
            low = mid + 1

    return None


my_list = [1, 3, 5, 7, 9]

print(binary_search(my_list, 3))   # 1
print(binary_search(my_list, -1))  # None


Почему это быстро

Обычный поиск проверяет элементы по одному.
Бинарный поиск каждый раз уменьшает область поиска в 2 раза.

Пример:

100 элементов → максимум около 7 шагов
1 000 элементов → максимум около 10 шагов
1 000 000 000 элементов → максимум около 30 шагов
Сложность

Временная сложность: O(log n)

Это значит, что с ростом размера массива количество операций растёт медленно.

Бинарный поиск намного быстрее линейного поиска O(n) на больших данных.

Что нужно запомнить
массив должен быть отсортирован
поиск идёт по индексам
сравнивается значение в середине массива
если элемент меньше среднего — идём влево
если больше — идём вправо
если равен — возвращаем индекс
Вывод

Бинарный поиск — это базовый и очень важный алгоритм, который показывает, как можно ускорить поиск за счёт правильной стратегии: не проверять всё подряд, а каждый раз отбрасывать половину данных.
